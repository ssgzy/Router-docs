---
title: パフォーマンス監視
description: サーバーのリアルタイムリソース使用状況の確認とシステムメンテナンス操作
---

サーバーのリアルタイムリソース使用状況を確認し、システムメンテナンス操作を実行します。Root アカウントでログイン後、システム設定（`/console/setting`）に移動して「パフォーマンス監視」タブを見つけてください。

パフォーマンス監視ページにはサーバーの現在の CPU 使用率、メモリ消費量、リクエスト量などのリアルタイム指標が表示されます。

## メンテナンス操作の実行

パフォーマンス監視ページで対応する操作ボタンを見つけてクリックすると、システムが即座に実行します：

| 操作 | 説明 | 使用場面 |
| --- | --- | --- |
| ディスクキャッシュのクリア | ディスクキャッシュ領域を解放 | ディスク使用量が高すぎる場合 |
| 統計のリセット | パフォーマンスカウンターをゼロにリセット | 統計を再開する必要がある場合 |
| GC の強制実行 | Go のガベージコレクションを手動でトリガーしてメモリを解放 | メモリ使用量が異常に高い場合 |
| ログファイルのクリーンアップ | 古いログファイルを削除してディスク領域を解放 | ログファイルが蓄積しすぎた場合 |

操作ボタンをクリックすると、ページに操作結果が表示され、指標データが更新されます。
