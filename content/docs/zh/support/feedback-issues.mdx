我们非常重视您的反馈，它们对 Code Router 的改进至关重要。如果您在使用过程中遇到任何问题，请通过 GitHub Issues 向我们反馈。

<Callout type="info" title="支持渠道说明">
  Code Router 目前处于早期阶段，官方支持渠道仍在搭建中。现阶段最可靠的反馈方式是源码仓库的 GitHub Issues；其他社区渠道（如交流群、邮件等）待补 / TBD。
</Callout>

## 📝 如何提交问题

请访问 Code Router 源码仓库的 Issues 页面：[Code Router Issues](https://github.com/ssgzy/Router-new-api/issues)

提交新的 issue 之前，建议您：

<Steps>
<Step>
搜索是否已经存在类似的问题或讨论。
</Step>
<Step>
确认使用的是最新版本，并尽量在最小可复现环境中重现问题。
</Step>
<Step>
查看 [常见问题文档](/zh/docs/support/faq) 中是否已有相关解答。
</Step>
</Steps>

## 💡 问题反馈模板

为帮助维护者快速定位问题，请尽量按以下模板填写。

### 报告问题的格式

```markdown
**例行检查**

[//]: # '方框内删除已有的空格，填 x 号'

- [ ] 我已确认目前没有类似 issue

- [ ] 我已完整查看过项目 README，尤其是常见问题部分
- [ ] 我理解并愿意跟进此 issue，协助测试和提供反馈

- [ ] 我确认本 issue 所描述的场景已遵守上游服务条款、平台规则、监管要求和项目合规政策

**问题描述**

**复现步骤**

**预期结果**

**实际结果**

**运行环境**
（部署方式、版本、所用模型分组 / 令牌分组等）

```

<Callout type="warn" title="注意事项">
  - 请严格按照模板格式提交 issue，并使用清晰的标题，方便其他用户搜索。
  - 请及时关注和回复您提交的 issue；不符合规范的 issue 可能会被直接关闭。
  - 如果问题已解决，请及时关闭 issue。
  - issue / PR / discussion 应符合项目合规政策和社区规则，维护者将按规则处理不符合要求的内容。
</Callout>
