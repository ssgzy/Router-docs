## 充值方式设置

"充值方式"用一段 JSON 数组描述展示给用户的支付按钮。下面是结构示例（其中的名称、颜色、类型与金额均为占位，请替换为实际配置）：

```json
[
  {
    "color": "rgba(var(--semi-blue-5), 1)",
    "name": "支付方式 A（占位）",
    "type": "channel_a"
  },
  {
    "color": "rgba(var(--semi-green-5), 1)",
    "name": "支付方式 B（占位）",
    "type": "channel_b",
    "min_topup": "（最低金额待补）"
  }
]
```

### 字段说明

- **name**: 展示文案，显示在"选择支付方式"的按钮上。
- **color**: 按钮/徽标的主题色或边框色。支持任意 CSS 颜色值，推荐使用现有设计令牌（如 `rgba(var(--semi-blue-5), 1)`）。
- **type**: 通道标识，用于后端路由与下单。不同取值会被映射到对应的支付网关或作为渠道参数透传。
  - （各 `type` 取值与所走网关的对应关系待补；具体逻辑见后端充值控制器，参考源码仓库：[Router-new-api](https://github.com/ssgzy/Router-new-api)。）
- **min_topup**: 该支付方式的最低充值金额（单位与页面货币一致）。当输入金额小于此值时，页面会提示并限制发起支付，后端也会校验。具体数值待补（TBD）。
- **排序**: 按数组顺序从左到右渲染。

## 充值金额配置

### 自定义充值数量选项

用于设置用户可在"选择充值额度"区域直接点击的预设金额。结构如下（数值为占位，实际金额待补）：

```json
["（额度 1 待补）", "（额度 2 待补）", "（额度 3 待补）"]
```

这些预设值会显示在充值页面，用户可直接点击选择，或自行输入金额。

### 充值金额折扣配置

可为不同充值金额配置折扣，键为充值金额，值为折扣率。结构如下（键值均为占位，实际折扣规则待补）：

```json
{
  "（充值金额待补）": "（折扣率待补，0-1 之间的小数）"
}
```

- **键**: 充值金额（字符串格式）。
- **值**: 折扣率（0–1 之间的小数，例如 `0.95` 表示按 95% 计费、即 5% 折扣）。
- 系统会根据配置自动计算实付金额与节省金额。
- 折扣功能的使用应符合授权范围、上游服务条款、平台规则与所在地法律法规要求。
- 具体实现逻辑见后端充值控制器，参考源码仓库：[Router-new-api](https://github.com/ssgzy/Router-new-api)。
