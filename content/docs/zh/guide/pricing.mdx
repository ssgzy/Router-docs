AI Router 是一个统一的、兼容 OpenAI 接口的 AI 网关。你只需一个 API Key 和一个 Base URL，即可调用多家上游模型供应商。本页说明计费的基本概念、在哪里查看价格、以及如何充值与查看用量。

<Callout type="warn" title="演示环境说明">
AI Router 目前为全新的演示项目，下文涉及的**具体价格、费率、货币金额、支付方式与充值额度均尚未最终确定**（待补 / TBD），请以你所部署的控制台中实际显示的数值为准。本页只描述计费的工作方式与操作位置，不代表任何真实价格承诺。
</Callout>

## 计费是如何工作的

AI Router 采用**按用量计费（pay-as-you-go）**的基本模型，核心要素如下：

- **按 Token 计费**：每次调用按消耗的输入 Token 与输出 Token 分别计价。不同模型、不同上游供应商的单价可能不同。
- **倍率（Rate）**：在按 Token 计费的基础上，可对某个模型或某个分组叠加倍率，用于统一调整价格。最终扣费约等于 `基础单价 × 倍率 × Token 数`。
- **模型分组（Model Group）与令牌分组（Token Group）**：管理员可以把模型归入不同分组，并为不同的令牌（API Key）指定可用分组，从而对不同用户群体应用不同的价格与权限。
- **额度 / 配额（Quota）**：账户余额以额度形式存在，调用时实时扣减。可对单个令牌设置剩余额度上限，也可设置每日调用配额等限制。

具体的单价、倍率数值与货币单位以控制台「定价」页面为准（费率 / 价格待补，TBD）。

## 在哪里查看价格

登录控制台后，可在「定价」或「模型」相关页面查看全站可用模型及其计费信息。

<Steps>

<Step>
### 打开控制台

通过你的 AI Router 部署地址访问控制台（演示占位地址：`https://ai.sammier.com`）。
</Step>

<Step>
### 进入定价 / 模型页面

在导航中找到「定价」（或「模型列表」）页面，即可看到当前可调用的模型清单。
</Step>

<Step>
### 查看每个模型的计费信息

页面会列出每个模型对应的**输入 / 输出单价**、所属**模型分组**以及适用的**倍率**。

> 截图占位：模型定价列表界面（待补 / TBD）。
</Step>

</Steps>

<Callout type="note">
不同令牌（API Key）可能被分配到不同的模型分组，因此你看到的可用模型与价格可能因账户配置而异。
</Callout>

## 额度与充值

账户通过额度进行扣费。当额度不足时，调用会被拒绝，需要先补充额度。

<Steps>

<Step>
### 查看当前余额 / 额度

在控制台的「钱包」或「额度」相关页面查看当前剩余额度与历史用量。
</Step>

<Step>
### 发起充值

进入充值页面，选择充值额度后提交。

<Callout type="warn">
具体的**充值额度档位、货币单位、价格与可用支付方式均待补（TBD）**，本演示环境未确认任何具体支付渠道。请以控制台实际提供的选项为准。
</Callout>
</Step>

<Step>
### 充值到账后开始调用

额度到账后即可使用你的 API Key 调用接口。OpenAI 兼容接入点示例：

```bash
curl https://ai.sammier.com/v1/chat/completions \
  -H "Authorization: Bearer $CODE_ROUTER_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "your-model-name",
    "messages": [{"role": "user", "content": "你好"}]
  }'
```
</Step>

</Steps>

## 用量与扣费记录

每次成功调用都会生成对应的用量与消费记录，便于对账：

- **用量记录**：记录模型名称、输入 / 输出 Token 数与本次调用的扣费额度。
- **消费记录**：按时间汇总扣费明细。
- **请求日志**：记录请求的路由与成功 / 失败状态，便于排查问题。

在控制台对应的「日志」或「账单」页面即可查看这些记录。

## 常见问题

<Accordions>

<Accordion title="一个 API Key 能调用所有模型吗？">
取决于该令牌被分配到的模型分组与权限。管理员可以限制某个令牌只能访问部分模型分组，因此并非所有 Key 都能调用全部模型。
</Accordion>

<Accordion title="价格和支付方式为什么都是“待补”？">
AI Router 当前是全新的演示项目，尚未确定真实价格、货币单位与支付渠道。文档中所有金额、费率与支付方式均为占位说明（TBD），最终以你部署的控制台实际显示为准。
</Accordion>

<Accordion title="额度用完会怎样？">
额度不足时调用会被拒绝并返回相应错误。补充额度后即可恢复调用。此外，单个令牌还可能受到剩余额度上限或每日配额等限制。
</Accordion>
