订阅是按计费周期购买的套餐,购买后在有效期内享受套餐内包含的配额或特权,适合有稳定用量需求的用户。在 Code Router 控制台左侧导航点击「订阅」,或直接访问 `/console/subscription`。

<Callout type="warn">
Code Router 目前为全新演示项目,本页所有价格、配额、计费周期、支付方式均为占位示例,**尚未最终确定**。文中标注「待补 / TBD」的位置以正式上线后的控制台实际显示为准。
</Callout>

## 订阅与按量计费的区别

Code Router 是 OpenAI 兼容的统一网关,你拿到一个 API Key 和一个 Base URL 即可调用多家模型供应商。计费上通常有两种方式:

<Cards>
  <Card title="按量计费" description="按实际消耗的 token 从余额中扣费,用多少扣多少,适合用量波动较大的场景。" />
  <Card title="订阅套餐" description="按周期一次性购买固定配额或特权,适合用量稳定、可预估的场景。" />
</Cards>

两者可以并存:订阅套餐内的配额优先使用,套餐额度用尽或过期后再回落到按量计费(具体回落规则待补 / TBD)。

## 查看订阅套餐

订阅套餐列表展示当前所有可购买的套餐,每个套餐通常包含以下信息:

- 套餐名称与说明
- 价格(金额待补 / TBD)
- 计费周期(可提供日、周、月等不同周期,具体可选周期待补 / TBD)
- 有效期
- 包含的配额 / 适用的模型分组、令牌分组(配额额度待补 / TBD)

<Callout type="info">
套餐与配额的对应关系基于 Code Router 的「模型分组(Model Group)」与「令牌分组(Token Group)」机制 —— 一个套餐可限定可调用的模型范围与配额上限。具体分组划分与额度由站点运营方配置,演示环境中以示例数据为准。
</Callout>

## 购买订阅

<Steps>
<Step>
在订阅页浏览可用套餐,对比各套餐的价格、有效期与包含配额,选择适合自己的套餐。
</Step>
<Step>
点击目标套餐的「购买 / 订阅」按钮,进入确认页核对套餐内容与金额(金额待补 / TBD)。
</Step>
<Step>
按页面提示完成支付。

<Callout type="warn">
支付方式与收款渠道尚未确定(支付方式待补 / TBD)。本演示不代表任何具体支付服务商已接入,也不展示任何真实金额或账户余额。
</Callout>
</Step>
<Step>
支付成功后返回订阅页,套餐立即生效,顶部会显示当前订阅状态。
</Step>
</Steps>

## 查看当前订阅状态

购买订阅后,在订阅页顶部可查看当前生效套餐的详细信息,通常包括:

- 套餐名称与有效期截止日期
- 套餐内剩余配额 / 已使用配额
- 适用的模型分组与令牌分组

<Callout type="info">
配额的实时消耗情况也可在控制台的「使用统计 / 用量记录」中查看,与你通过 `https://api.code-router.example/v1` 发起的实际调用一一对应。
</Callout>

## 自动续费

订阅详情中可设置到期后是否自动续费:

- **开启自动续费**:套餐到期时按当前套餐自动续订,保证配额不中断(续费扣费规则待补 / TBD)。
- **关闭自动续费**:套餐到期后停止续订,到期即回落到按量计费或暂停服务(具体行为待补 / TBD)。

<Callout type="warn">
由于支付方式与价格尚未最终确定,自动续费的实际扣费时机、扣费金额与失败重试策略均待补(TBD),以正式上线后的控制台说明为准。
</Callout>

## 常见问题

<Accordions>
<Accordion title="套餐里的配额怎么计算?">
配额基于模型分组与令牌分组,按 token 用量扣减。不同模型的折算比例可能不同,具体折算规则与额度待补(TBD)。
</Accordion>
<Accordion title="套餐到期后还能用吗?">
到期后套餐内的剩余配额是否保留、是否回落到按量计费,取决于站点运营方的策略(待补 / TBD)。建议开启自动续费以避免服务中断。
</Accordion>
<Accordion title="价格和支付方式什么时候确定?">
Code Router 仍处于演示阶段,价格、计费周期与支付方式均未最终敲定。如需反馈或了解进展,可前往源码仓库 <a href="https://github.com/ssgzy/Router-new-api">github.com/ssgzy/Router-new-api</a> 的 Issues 提交问题。
</Accordion>
