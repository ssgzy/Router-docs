令牌（Token）是调用 Code Router API 的凭证。Code Router 提供 OpenAI 兼容接口，你用同一个令牌 + 同一个 Base URL 即可访问平台聚合的多家模型供应商。每个令牌都可以独立配置权限范围、模型限制与配额上限，便于按用途隔离。左侧导航点击「令牌」，或直接访问 `/console/token`。

![令牌列表页](/assets/guide/feature-guide/token-list.png)

令牌列表展示所有已创建的令牌，包含名称、状态、已用配额、剩余配额、过期时间等信息。

## 创建令牌

<Steps>

<Step>

### 打开创建弹窗

在令牌列表页点击右上角「创建令牌」按钮，弹出创建弹窗。

</Step>

<Step>

### 填写令牌名称

建议按用途命名（如「生产环境」「测试用」「CI 流水线」），便于后续在用量记录中区分来源。

![创建令牌弹窗](/assets/guide/feature-guide/token-create.png)

</Step>

<Step>

### 配置高级选项

按需配置以下选项：

| 配置项 | 说明 |
| --- | --- |
| 过期时间 | 设置有效期，留空或设为 `-1` 表示永不过期 |
| 剩余配额 | 限制该令牌可消耗的最大配额，超出后自动失效 |
| 无限配额 | 开启后该令牌不单独限额（仍受账户总配额约束） |
| 模型限制 | 限定该令牌只能调用指定模型，留空表示不限制 |
| IP 白名单 | 限定允许使用该令牌的来源 IP，留空表示不限制 |
| 分组 | 指定该令牌使用的渠道分组（不同分组可对应不同供应商/价格策略） |

</Step>

<Step>

### 保存令牌 Key

点击「提交」，弹窗会显示完整的令牌 Key，**请立即复制保存**。关闭弹窗后将无法再次查看完整 Key。

![令牌创建成功，显示完整 Key](/assets/guide/feature-guide/token-created-key.png)

</Step>

</Steps>

<Callout type="warn">
  令牌 Key 仅在创建时完整显示一次，请立即复制保存。令牌 Key 具有完整的 API 调用权限，请勿泄露给他人，也不要提交到代码仓库或前端代码中。
</Callout>

## 使用令牌调用 API

创建令牌后，将它作为 `Authorization: Bearer <你的令牌 Key>` 传入，并把请求地址指向 Code Router 的 OpenAI 兼容端点：

- Base URL：`https://ai.sammier.com`
- OpenAI 兼容端点：`https://ai.sammier.com/v1`

```bash
curl https://ai.sammier.com/v1/chat/completions \
  -H "Authorization: Bearer <你的令牌 Key>" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "<模型名称>",
    "messages": [{"role": "user", "content": "你好"}]
  }'
```

<Callout type="info">
  任何兼容 OpenAI 协议的 SDK / 客户端都可使用：把官方 Base URL 改为 `https://ai.sammier.com/v1`，并将 API Key 替换为你的令牌 Key 即可。具体可调用的模型名称以「模型」页面或 `GET /v1/models` 返回为准。
</Callout>

## 编辑令牌

在令牌列表中找到目标令牌，点击右侧「编辑」按钮，可修改令牌的配置选项（过期时间、配额、模型限制、IP 白名单、分组等），但不包括令牌 Key 本身。Key 一经生成不可更改；若需轮换，请新建令牌并停用旧令牌。

## 删除令牌

在令牌列表中找到目标令牌，点击右侧「删除」按钮，确认后该令牌立即失效且无法恢复。请确保没有线上服务仍在使用该令牌后再删除。
