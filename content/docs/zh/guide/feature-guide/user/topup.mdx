配额是 AI Router 的内部计费单位。每次调用模型的配额消耗量大致为：实际 Token 数 × 该模型的倍率（即模型分组 / 令牌分组所配置的费率）。在控制台左侧导航点击「钱包管理」，或直接访问 `/console/topup`，即可进入充值页面。

<Callout type="warn">
本页所列的充值方式、价格、费率与金额均为 **演示占位**，尚未在本演示环境中最终确定。**（价格 / 费率 / 支付方式待补 — TBD）**。请勿将任何金额或支付渠道视为真实可用。
</Callout>

## 配额与倍率如何工作

- **配额**：账户余额以配额为单位记账，调用模型时按 Token 用量扣减。
- **模型倍率**：不同模型按其分组配置的倍率换算配额消耗，倍率越高，单位 Token 消耗的配额越多。
- **令牌分组 / 模型分组**：管理员可为不同分组配置不同费率，从而对不同模型或不同用户群体差异化计价。

具体倍率数值、各模型的单位价格、以及「每百万 Token 多少费用」等定价表 **（费率 / 价格待补 — pricing TBD）**，将在正式环境中由管理员配置后公布。

## 充值方式

AI Router 计划支持通过 **在线支付** 和 **兑换码** 两种方式增加账户配额。

<Callout type="info">
下表中的具体支付渠道为占位示意，**（可用支付方式待补 — TBD）**。本演示不保证任何一种支付方式当前可用。
</Callout>

| 充值方式 | 说明 |
| --- | --- |
| 兑换码 | 输入管理员生成的兑换码，直接增加配额 |
| 在线支付 | 通过线上支付渠道充值（具体渠道与费用待补 — TBD） |

## 在线支付充值

<Steps>

<Step>
### 选择充值金额

在充值页选择预设的充值金额，或手动输入自定义金额。具体金额档位与计价 **（金额 / 价格待补 — TBD）**。
</Step>

<Step>
### 选择支付方式并发起支付

选择一种在线支付方式（**具体渠道待补 — TBD**），点击「充值」按钮，页面会跳转至对应的支付渠道。
</Step>

<Step>
### 完成支付

在支付渠道完成付款后，会自动跳回 AI Router，账户配额余额随之更新。
</Step>

</Steps>

## 兑换码充值

<Steps>

<Step>
### 输入兑换码

在充值页找到「兑换码」输入区域，在输入框中粘贴或输入管理员提供的兑换码。
</Step>

<Step>
### 完成兑换

点击「兑换」按钮，系统会验证兑换码的有效性。兑换成功后，页面会提示获得的配额数量，账户余额同步更新。
</Step>

</Steps>

## 邀请返利

每个账号都有唯一的邀请码。当其他人使用你的邀请码注册成功时，你即可获得邀请奖励。

<Callout type="warn">
邀请奖励的具体配额数额由管理员配置，**（奖励金额待补 — TBD）**，演示环境中不代表真实数值。
</Callout>

### 获取邀请码

在充值页或个人设置页找到「邀请」区域，复制你的专属邀请码。

### 邀请奖励机制

1. **注册即返利**：被邀请人使用你的邀请码注册成功后，你（邀请人）会获得系统配置的一笔「邀请奖励配额」（一次性到账，具体数额待补 — TBD）。
2. **转入主余额**：点击「转入余额」按钮，即可将累计的返利配额转入你的账户主余额中使用。
