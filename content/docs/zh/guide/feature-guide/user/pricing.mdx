Code Router 是一个统一的、兼容 OpenAI 协议的 AI 网关：你用一个 API Key 和一个 Base URL 即可调用多家模型供应商。本页介绍如何查看模型定价、计费如何换算，以及分组与充值的概念。

</Callout>

## 查看定价

### 访问定价页

点击左侧导航栏的「定价」，或直接访问站点下的 `/pricing` 路径。

### 浏览模型价格

定价页会列出当前可用的模型，每一行通常包含：

- 模型名称
- 所属模型分组
- 输入价格（每单位 Token 的计费占位，**单价待补 / TBD**）
- 输出价格（每单位 Token 的计费占位，**单价待补 / TBD**）

### 搜索模型

在页面顶部的搜索框中输入模型名称关键词，可快速定位特定模型，查看其计费信息。

## 计费规则

作为兼容 OpenAI 协议的网关，Code Router 按调用产生的 Token 用量计费，输入与输出通常分别计价。

<Steps>

<Step>
**统计 Token 用量**：每次请求结束后，网关会记录该次调用的输入 Token 数与输出 Token 数（即 OpenAI 协议中的 `prompt_tokens` 与 `completion_tokens`）。
</Step>

<Step>
**按单价换算消耗**：实际扣减 ≈ 输入 Token 数 × 输入单价 + 输出 Token 数 × 输出单价。

具体的计费粒度（例如每多少 Token 为一档）与单价数值**待补 / TBD**。
</Step>

<Step>
**从配额 / 余额中扣减**：换算出的消耗会从你的账户配额或余额中扣除，并记录到用量明细中，便于事后核对。
</Step>

</Steps>

<Callout type="info">
输出 Token 的单价通常高于输入 Token，这是大多数兼容 OpenAI 的网关的常见做法。Code Router 的具体差异比例**待补 / TBD**。
</Callout>

## 分组与倍率

Code Router 支持「模型分组」「令牌分组」等机制，不同分组可以对应不同的可用模型集合与计费倍率。

- **模型分组**：决定一个 Key 能访问哪些模型。
- **令牌分组 / 用户分组**：可对计费应用不同的倍率，例如某些分组享有更低或更高的折扣。
- 同一个模型在不同分组下的最终单价可能不同，**具体倍率待补 / TBD**。

最终扣减以平台实际计算为准，你可以在用量记录或充值页查看每次调用产生的消耗。

## 充值与额度

<Callout type="warn">
Code Router 演示阶段尚未确定任何具体的充值方式、支付渠道、币种或套餐。下文仅说明通用概念，**充值方式与金额待补 / TBD**，请勿将其视为已上线的真实支付能力。
</Callout>

- **额度 / 配额**：账户的可用额度用于支付调用产生的消耗。额度耗尽后，相关调用会被拒绝。
- **充值入口**：通常位于「充值」或「钱包」页面（演示阶段入口与流程**待补 / TBD**）。
- **支付方式**：演示阶段**未提供**任何具体支付渠道，相关说明**待补 / TBD**。
- **余额查询**：可在充值页或用量明细中查看当前余额及其变化（金额数值**待补 / TBD**）。

## 接入端点

无论使用哪种计费分组，调用方式都遵循 OpenAI 兼容协议：

- Base URL：`https://api.code-router.example`
- OpenAI 兼容端点：`https://api.code-router.example/v1`

将上述 Base URL 与你的 API Key 配置到任意兼容 OpenAI 的客户端或 SDK 即可发起调用，产生的用量会按本页规则计费。
