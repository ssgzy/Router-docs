在 Code Router 中创建和管理订阅套餐，控制哪些套餐对用户可见，以及为用户手动开通订阅。使用管理员账号登录后，直接访问 `/console/subscription`（管理员视角与用户购买界面不同）。

<Callout type="info">
本页面介绍的为通用操作流程。Code Router 目前仍处于演示阶段，下文涉及的**价格、配额数值、有效期、付款方式**等均**尚未最终确定**，文中以「待补 / TBD」标注，请以你实例中的实际配置为准。
</Callout>

套餐列表展示所有已创建的订阅套餐，包含名称、价格、有效期、状态（上架 / 下架）等信息。

## 套餐与配额的关系

在 Code Router 这类 OpenAI 兼容网关中，订阅套餐本质上是把一定量的**配额（quota）**与一段**有效期**打包，便于用户一次性获取调用额度。

- **配额**用于扣减用户调用模型时产生的消耗，具体计费规则与单价由模型分组 / 令牌分组的费率决定（**费率 / 单价待补 TBD**）。
- **有效期**决定套餐绑定后多久过期；过期后剩余配额的处理方式取决于你的实例配置。
- 套餐价格、计费币种以及可选的付款方式均**尚未在本演示中确定**（**金额 / 付款方式待补 TBD**），本文不对任何具体付款渠道做出承诺。

## 创建套餐

<Steps>

<Step>
在订阅管理页点击「创建套餐」按钮，弹出创建弹窗。
</Step>

<Step>
填写以下信息：

- **套餐名称**：用户购买时看到的名称
- **价格**：套餐售价（**金额 / 币种待补 TBD**）
- **有效期类型**：选择日 / 周 / 月 / 自定义天数
- **包含配额**：套餐内包含的配额数量（**数值待补 TBD**）
</Step>

<Step>
点击「提交」，套餐创建完成并默认处于下架状态。
</Step>

</Steps>

<Callout type="warn">
请勿在文档或对外页面中填写未经确认的真实价格、付款渠道或金额。演示阶段建议先以占位值创建套餐，待运营信息确定后再调整。
</Callout>

## 上架 / 下架套餐

<Steps>

<Step>
在套餐列表中找到目标套餐，点击右侧「上架」或「下架」按钮。
</Step>

<Step>
上架后用户在订阅页可看到并购买该套餐；下架后套餐对用户不可见，但已购买的订阅不受影响。
</Step>

</Steps>

## 为用户手动开通订阅

<Steps>

<Step>
在订阅管理页点击「手动绑定」按钮。
</Step>

<Step>
输入目标用户名或邮箱，选择要开通的套餐。
</Step>

<Step>
点击「确认」，系统为该用户创建订阅记录，立即生效。
</Step>

</Steps>

手动开通适用于内测授权、补偿发放等场景，可在不经过付款流程的情况下直接为用户绑定套餐配额。
