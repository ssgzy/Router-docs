## 关于 Code Router

Code Router 是一个统一的、兼容 OpenAI 协议的 AI 网关 / 中转层。用户只需一个 API Key 和一个 Base URL，即可调用多家模型供应商，并借助模型分组、令牌分组、配额与用量统计来集中管理调用。

对企业与团队而言，这种统一接入层可以降低内部接入与治理的复杂度：无需为每个供应商分别维护凭证与计费口径，而是通过单一入口完成路由、配额控制和用量审计。

<Callout type="info">
  Code Router 是一个全新的演示项目（基于开源项目二次开发，源码见
  [GitHub 仓库](https://github.com/ssgzy/Router-new-api)）。本页中涉及价格、套餐、SLA、联系方式等具体运营信息均尚未确定，相关位置以「待补 / TBD」标注。
</Callout>

## 企业场景能做什么

下面是任意一个兼容 OpenAI 协议的网关在企业内部通常可以承担的通用能力，供参考：

<Cards>
  <Card title="统一接入" href="https://github.com/ssgzy/Router-new-api">
    用一个 Base URL 和一个 API Key 调用多家供应商，屏蔽上游差异。
  </Card>
  <Card title="分组与配额">
    通过模型分组、令牌分组设置不同团队 / 项目的可用模型与额度上限。
  </Card>
  <Card title="用量可观测">
    记录每次调用的用量，便于内部成本核算与审计。
  </Card>
  <Card title="集中治理">
    在单一入口完成路由、限额与凭证管理，降低维护成本。
  </Card>
</Cards>

## 接入方式

任何兼容 OpenAI 协议的客户端或 SDK 都可以直接接入，只需替换 Base URL 与 API Key。

<Steps>
  <Step>
    将客户端的 Base URL 指向 Code Router 的兼容端点：`https://ai.sammier.com/v1`
  </Step>
  <Step>
    使用在 Code Router 中创建的 API Key 作为 `Authorization: Bearer <API_KEY>`。
  </Step>
  <Step>
    在 Code Router 控制台中为团队配置模型分组、令牌分组与配额。
  </Step>
  <Step>
    通过用量统计核对调用情况（计费口径与单价待补）。
  </Step>
</Steps>

示例：列出可用模型

```bash
curl https://ai.sammier.com/v1/models \
  -H "Authorization: Bearer <API_KEY>"
```

## 合规与授权

Code Router 致力于为开发者与企业提供合法授权场景下的统一 API 网关与用量管理能力。使用与合作应基于以下原则：

- 合作与使用应在**合法授权、合规部署和正当业务场景**下开展。
- 需遵守上游服务条款、平台规则和监管要求。
- 客户必须合法取得上游模型 / 服务的授权，并对自身服务资质、合规运营、内容安全和用户管理承担责任。
