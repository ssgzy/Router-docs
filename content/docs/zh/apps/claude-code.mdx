<Callout type="info" title="本页说明">
  Claude Code 是 Anthropic 推出的终端编程助手。它通过环境变量读取接入点地址与密钥，因此可以将其指向
  **Code Router** —— 一个 OpenAI / Anthropic 兼容的统一 AI 网关。配置完成后，你只需一个 Code Router
  API Key 和一个 Base URL，即可在 Claude Code 中调用网关后端接入的各家模型。

  - Claude Code 官方文档：[https://www.anthropic.com/claude-code](https://www.anthropic.com/claude-code)
  - Code Router 源码仓库：[https://github.com/ssgzy/Router-new-api](https://github.com/ssgzy/Router-new-api)
</Callout>

## 工作原理

Claude Code 默认调用 Anthropic 官方接口。它支持通过两个环境变量改写请求目标：

- `ANTHROPIC_BASE_URL` —— Anthropic 兼容接口的基础地址。指向 Code Router 后即由网关接管转发。
- `ANTHROPIC_AUTH_TOKEN` —— 鉴权令牌。填入你在 Code Router 中创建的 API Key（形如 `sk-...`）。

<Callout type="warn" title="重要：改写 Base URL 后的行为">
  设置 `ANTHROPIC_BASE_URL` 后，Claude Code 的全部请求（包括其默认/预设模型）都会改走你配置的接入点，
  而不再消耗 Anthropic 官方账号额度。用量与计费以 Code Router 一侧的记录为准。
</Callout>

## 准备工作

在配置 Claude Code 之前，请先在 Code Router 控制台准备好两样东西：

<Steps>
  <Step>
    **获取 Base URL。** 使用你的 Code Router 部署地址。本文档统一以演示占位地址
    `https://ai.sammier.com` 表示；OpenAI 兼容端点为 `https://ai.sammier.com/v1`。
    请替换为你实际的部署域名。
  </Step>
  <Step>
    **创建 API Key。** 在 Code Router 控制台的令牌（Token）管理中新建一个密钥，复制形如 `sk-...`
    的字符串。该密钥用于鉴权，并关联到对应的模型分组与配额。
  </Step>
</Steps>

## 安装 Claude Code

Claude Code 跨平台支持 Windows、macOS 与 Linux。

<Tabs items={["macOS / Linux", "Windows"]}>
  <Tab value="macOS / Linux">
    在终端运行官方安装脚本：

    ```bash
    curl -fsSL https://claude.ai/install.sh | bash
    ```

    如果安装后提示需要把可执行目录加入 `PATH`，按提示执行类似命令（按你的 shell 选择 `.bashrc` 或 `.zshrc`）：

    ```bash
    echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc
    ```
  </Tab>
  <Tab value="Windows">
    Claude Code 依赖 Node.js 环境，且在 Windows 上建议通过 Git Bash 安装。

    1. 安装 [Node.js](https://nodejs.org/) LTS 版本（保持默认设置即可）。
    2. 安装 [Git for Windows](https://git-scm.com/downloads/win)（提供 Git Bash）。
    3. 打开 PowerShell，执行：

       ```bash
       npm install -g @anthropic-ai/claude-code
       ```

    <Callout type="warn" title="Windows 注意事项">
      - 建议使用 PowerShell 而非 CMD。
      - 遇到权限问题时，尝试以管理员身份运行。
      - 个别杀毒软件可能误报，必要时将其加入白名单。
    </Callout>
  </Tab>
</Tabs>

安装完成后，验证版本：

```bash
claude --version
```

显示版本号即表示安装成功。

## 接入 Code Router

把 Claude Code 的接入点指向 Code Router，只需设置两个环境变量。将下面的占位值替换为你自己的 Base URL 与 API Key。

<Tabs items={["macOS / Linux", "Windows (PowerShell)"]}>
  <Tab value="macOS / Linux">
    临时生效（仅当前终端会话）：

    ```bash
    export ANTHROPIC_BASE_URL="https://ai.sammier.com"
    export ANTHROPIC_AUTH_TOKEN="sk-..."
    ```

    持久生效（写入 shell 配置文件，按你的 shell 选择 `.zshrc` 或 `.bashrc`）：

    ```bash
    echo 'export ANTHROPIC_BASE_URL="https://ai.sammier.com"' >> ~/.zshrc
    echo 'export ANTHROPIC_AUTH_TOKEN="sk-..."' >> ~/.zshrc
    source ~/.zshrc
    ```
  </Tab>
  <Tab value="Windows (PowerShell)">
    临时生效（仅当前 PowerShell 会话）：

    ```powershell
    $env:ANTHROPIC_BASE_URL = "https://ai.sammier.com"
    $env:ANTHROPIC_AUTH_TOKEN = "sk-..."
    ```

    持久写入用户环境变量：

    ```powershell
    [Environment]::SetEnvironmentVariable('ANTHROPIC_BASE_URL', 'https://ai.sammier.com', 'User')
    [Environment]::SetEnvironmentVariable('ANTHROPIC_AUTH_TOKEN', 'sk-...', 'User')
    ```

    设置后请重新打开 PowerShell，使变量生效。
  </Tab>
</Tabs>

<Callout type="info" title="验证变量已生效">
  ```bash
  echo $ANTHROPIC_BASE_URL   # macOS / Linux
  ```

  ```powershell
  $env:ANTHROPIC_BASE_URL    # Windows PowerShell
  ```

  应输出你配置的 Code Router Base URL。
</Callout>

## 开始使用

在任意项目目录下启动 Claude Code：

```bash
# 进入你的项目目录
cd /path/to/your/project

# 启动
claude
```

如需切换模型，在交互界面中输入：

```bash
/model
```

按回车进入选择列表。可用模型取决于你的 Code Router 部署所接入的供应商及当前 API Key 所属模型分组——请以控制台中的模型列表为准。

## 常见问题

<Accordions>
  <Accordion title="环境变量不生效 / 请求仍走官方接口">
    - 确认修改的是当前 shell 实际加载的配置文件（`.zshrc` 或 `.bashrc`）。
    - 重新打开终端，或执行 `source ~/.zshrc`（Windows 则重开 PowerShell）。
    - 用 `echo $ANTHROPIC_BASE_URL` 核对取值是否为你的 Code Router 地址。
  </Accordion>
  <Accordion title="鉴权失败 / 401">
    - 检查 `ANTHROPIC_AUTH_TOKEN` 是否为有效的 Code Router API Key（形如 `sk-...`），是否被复制完整、无多余空格。
    - 在 Code Router 控制台确认该密钥处于启用状态，且其模型分组包含你要调用的模型。
  </Accordion>
  <Accordion title="提示找不到 claude 命令">
    - 安装脚本通常会把可执行文件放在 `~/.local/bin`，确认该目录已加入 `PATH`。
    - macOS / Linux：`echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc`。
  </Accordion>
  <Accordion title="macOS 安全设置阻止运行">
    打开“系统设置” → “隐私与安全性”，在被拦截的提示处点击“仍要打开 / 允许”。
  </Accordion>
  <Accordion title="Linux 缺少依赖库">
    ```bash
    # Ubuntu / Debian
    sudo apt install build-essential

    # CentOS / RHEL
    sudo dnf groupinstall "Development Tools"
    ```
  </Accordion>
</Accordions>
