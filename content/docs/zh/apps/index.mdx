## 概述

AI Router 是一个兼容 OpenAI 接口的统一 AI 网关。任何支持「自定义 OpenAI 兼容接口」的应用，只要填入一个 API 地址和一个 API Key，就能通过 AI Router 访问多家模型供应商。

下面列出了常见的桌面客户端、命令行工具和聊天机器人框架，并为每个应用提供了独立的配置指南。

## 通用接入步骤

大多数应用的配置方式是一致的。在应用的「模型供应商 / API 设置」中选择 **OpenAI 兼容（Custom OpenAI / OpenAI-Compatible）** 渠道，然后填入以下信息：

<Steps>

<Step>

### 准备 API Key

在 AI Router 控制台创建一个 API 令牌（Token），并复制以 `sk-` 开头的密钥。建议为不同应用分别创建令牌，便于单独管理配额与用量。

</Step>

<Step>

### 填写 API 地址

将应用中的「API 地址 / Base URL」设置为 AI Router 的接口地址：

```text
https://ai.sammier.com/v1
```

部分应用只需要填写主域名 `https://ai.sammier.com`，会自动补全 `/v1` 路径；如果连接失败，请尝试带或不带 `/v1` 两种写法。

</Step>

<Step>

### 选择模型

在模型列表中填写或选择需要调用的模型名称。可用的模型由你所在的「模型分组 / 令牌分组」决定，具体清单请以 AI Router 控制台展示为准。

</Step>

<Step>

### 测试连接

保存配置后发送一条测试消息。若返回正常回复，则接入成功；随后可在 AI Router 控制台的用量记录中查看本次调用。

</Step>

</Steps>

### 命令行 / 环境变量接入

对于 CLI 工具和脚本，通常通过环境变量配置 OpenAI 兼容接口：

```bash
export OPENAI_BASE_URL="https://ai.sammier.com/v1"
export OPENAI_API_KEY="sk-你的令牌"
```

不同工具使用的变量名可能不同（如 `OPENAI_API_BASE`、`ANTHROPIC_BASE_URL` 等），请参考对应应用的详细文档。

<Callout type="info">
模型分组、令牌分组与配额的具体规则，以及各模型的计费方式，请以 AI Router 控制台中的实际展示为准（定价细则待补）。
</Callout>
