<Callout type="info">
  🔀 CC Switch 是一款开源、跨平台的 AI CLI 统一管理工具，支持 Claude Code、Codex 和 Gemini CLI
  的 Provider 配置一键切换、MCP 服务器统一管理、系统提示词（Prompts）管理以及 Skills 扩展管理，
  让你在多个 AI 编程助手之间自由切换，无需手动编辑配置文件。
</Callout>

Code Router 是一个统一的、兼容 OpenAI 接口的 AI 网关（基于开源项目二次开发）。注册后你会获得**一个 API Key** 与**一个 Base URL**，即可通过它调用多家模型供应商。本页介绍如何在 CC Switch 中创建一个指向 Code Router 的 Provider，从而为 Claude Code / Codex / Gemini CLI 统一供给模型。

- GitHub 仓库（CC Switch）：[https://github.com/farion1231/cc-switch](https://github.com/farion1231/cc-switch)
- 下载地址：[GitHub Releases](https://github.com/farion1231/cc-switch/releases)
- Code Router 源码仓库：[https://github.com/ssgzy/Router-new-api](https://github.com/ssgzy/Router-new-api)

## 核心特性

### 🔌 Provider 管理
- **一键切换** — 在 Claude Code、Codex、Gemini 的 API 配置之间一键切换，无需手动修改环境变量或配置文件
- **多端点支持** — 每个 Provider 可配置多个端点，便于在不同网关之间切换
- **多级模型配置** — 可分别为不同档位（如轻量 / 均衡 / 旗舰）指定模型

### 🛠️ MCP 服务器管理
- **跨应用统一管理** — 单面板管理 Claude / Codex / Gemini 三端的 MCP 服务器
- **多种传输类型** — 支持 stdio、HTTP、SSE（Server-Sent Events）
- **导入导出** — 统一导入导出，便于在不同设备间迁移配置

### 💬 Prompts 管理
- **多预设系统提示词** — 无限预设、快速切换
- **跨应用支持** — Claude（`CLAUDE.md`）、Codex（`AGENTS.md`）、Gemini（`GEMINI.md`）

### 🌐 多平台支持
- **桌面应用** — Windows、macOS、Linux 原生安装包
- **Web 版本** — 适用于无头服务器 / SSH 远程环境的浏览器访问方案
- **CLI 版本** — 命令行交互模式与命令模式

## 准备：获取 Code Router 凭据

在 CC Switch 中添加 Provider 前，先在 Code Router 控制台准备两项信息：

- **Base URL（API 地址）**：`https://ai.sammier.com`
  CC Switch 中部分应用要求填写到 `/v1`（兼容 OpenAI 协议）：`https://ai.sammier.com/v1`
- **API Key（API 密钥）**：在 Code Router 控制台「令牌管理」中创建并复制，形如 `sk-...`

<Callout type="warn">
  请使用你自己部署的 Code Router，或确认服务方具备合法上游授权与合规义务。不要将来源不明的 API 地址或密钥接入生产环境。
</Callout>

## 在 CC Switch 中接入 Code Router

<Steps>

<Step>
### 打开 Provider 管理并新增配置

在 CC Switch 中切换到目标应用（**Claude** / **Codex** / **Gemini**），新增一个 Provider 配置，并为其填写一个便于识别的**名称**（例如 `Code Router`）。
</Step>

<Step>
### 填写 Base URL 与 API Key

将 **Base URL** 设为 Code Router 的接口地址，将 **API Key** 设为你的 Code Router 密钥：

<Tabs items={['Anthropic 兼容（Claude Code）', 'OpenAI 兼容（Codex 等）']}>

<Tab value="Anthropic 兼容（Claude Code）">
- Base URL：`https://ai.sammier.com`
- API Key：`sk-...`（你的 Code Router 令牌）
</Tab>

<Tab value="OpenAI 兼容（Codex 等）">
- Base URL：`https://ai.sammier.com/v1`
- API Key：`sk-...`（你的 Code Router 令牌）
</Tab>

</Tabs>
</Step>

<Step>
### 指定模型

在配置中填写要使用的模型名称（可分档设置主力模型与轻量模型）。具体可用模型以你的 Code Router 控制台中模型 / 令牌分组所授权的为准（可用模型列表待补）。
</Step>

<Step>
### 切换并运行

保存后，在 CC Switch 中将当前 Provider 切换为刚创建的 **Code Router** 配置，CC Switch 会自动写入对应 CLI 的配置文件 / 环境变量。随后直接启动你的 CLI（如 `claude`、`codex`、`gemini`）即可通过 Code Router 调用模型。
</Step>

</Steps>

<Callout type="info">
  如需校验连通性，可在 CLI 中发起一次简单对话；若返回正常说明 Base URL 与 API Key 配置成功。也可直接对端点 `https://ai.sammier.com/v1/models` 发起请求，确认凭据有效。
</Callout>

## 手动配置（不使用 CC Switch）

如果你不想通过 CC Switch，也可以直接为对应 CLI 设置环境变量。以兼容 OpenAI 协议的 CLI 为例：

```shell
export OPENAI_BASE_URL="https://ai.sammier.com/v1"
export OPENAI_API_KEY="sk-..."   # 你的 Code Router 令牌
```

具体环境变量名称以各 CLI 的文档为准；核心是将 **Base URL 指向 Code Router、API Key 设为你的 Code Router 令牌**。

## 安装 CC Switch

### macOS（推荐 Homebrew）

```shell
brew tap farion1231/ccswitch
brew install --cask cc-switch
```

### Windows

从 [Releases](https://github.com/farion1231/cc-switch/releases) 下载 `.msi` 安装包或便携版 `.zip`。

### Linux

从 [Releases](https://github.com/farion1231/cc-switch/releases) 下载 `.deb` 包或 `.AppImage`。

ArchLinux 用户：

```shell
paru -S cc-switch-bin
```

### Web 版本（无头 / SSH 服务器）

```shell
wget https://github.com/farion1231/cc-switch/releases/latest/download/cc-switch-web-linux-x64.tar.gz
tar -xzf cc-switch-web-linux-x64.tar.gz
cd cc-switch-web/
./cc-switch-web
```

默认端口 `17666`，通过浏览器访问 `http://localhost:17666`。
