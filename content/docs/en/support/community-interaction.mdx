## Community guidelines

To keep the community useful and welcoming, please follow these principles:

1. **Be respectful.** Maintainers and other community members participate voluntarily and are not obligated to provide support on demand.
2. **Stay on topic.** Keep discussion focused on the use, deployment, development, and feedback of the Code Router open-source project.
3. **No account trading or solicitation.** Do not post offers to buy or sell accounts, keys, or quotas, and do not solicit such transactions.
4. **Respect the license.** Code Router is distributed under an open-source license; use it in accordance with that license. See [Project Introduction](/en/docs/guide/wiki/basic-concepts/project-introduction).
5. **Comply with the law.** When deploying or using the project, follow applicable laws and regulations, upstream provider terms of service, and content-safety requirements.
6. **Use legal authorization.** Discussions should center on compliant deployment and legitimate engineering questions.

<Callout title="Reporting bugs and requesting features">
  For feature suggestions or bug reports, open an issue on the source repository's [GitHub Issues](https://github.com/ssgzy/Router-new-api/issues). This makes problems easier to track, reproduce, and resolve than ad-hoc messages.
</Callout>

## Filing a good issue

A clear report gets a faster answer.

<Steps>
  <Step>
    **Search first.** Check whether your question or bug has already been raised.
  </Step>
  <Step>
    **Describe the setup.** Note your deployment method, version (if known), and the model provider or model group involved.
  </Step>
  <Step>
    **Show the request.** Include the base URL you call (for example `https://api.code-router.example/v1`), the endpoint, and a minimal example — but **never paste real API keys, tokens, or secrets**.
  </Step>
  <Step>
    **Include the error.** Paste the exact error message or response, the steps to reproduce, and what you expected to happen.
  </Step>
</Steps>

<Callout type="warn" title="Protect your credentials">
  Redact API keys, tokens, and any other secrets before sharing logs or requests publicly.
</Callout>

Code Router is open source. Contributions — bug fixes, documentation improvements, and features — are welcome through the [source repository](https://github.com/ssgzy/Router-new-api). Open an issue to discuss larger changes before submitting a pull request.
