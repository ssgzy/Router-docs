AI Router is a unified, OpenAI-compatible AI gateway. You get one API key and one base URL to call many model providers, with model groups, token groups, quotas, and usage tracking. The questions below cover the most common topics.

  AI Router is a brand-new demo. Anything involving prices, payment, contacts, or other operational specifics is still to be determined (TBD) and is noted as such below.
</Callout>

## Getting started

<Accordions>
  <Accordion title="What is AI Router?">
    AI Router is an OpenAI-compatible AI gateway (a relay/aggregator). Instead of integrating each model provider separately, you point your existing OpenAI-compatible client at AI Router using a single base URL and a single API key, and AI Router routes the request to the appropriate upstream provider channel.
  </Accordion>

  <Accordion title="How do I get an API key?">
    Sign in to the AI Router console, open the token/key management page, and create a new token. The generated key is what you place in your client's `Authorization: Bearer <key>` header. Keep it secret; treat it like a password.
  </Accordion>

  <Accordion title="What base URL do I use?">
    Use your AI Router deployment's base URL. For an OpenAI-compatible client, the endpoint is:

    ```
    https://ai.sammier.com/v1
    ```

    Set this as the API base (sometimes called "base URL", "API host", or "endpoint") in your client, and use your AI Router token as the API key. Most clients append paths such as `/chat/completions` automatically.
  </Accordion>

  <Accordion title="Which endpoints are supported?">
    AI Router exposes OpenAI-compatible endpoints under `/v1`, including model listing and chat completions (streaming and non-streaming). Exact endpoint coverage depends on what the configured upstream providers support. If your client speaks the OpenAI API, it should work without code changes — only the base URL and key change.
  </Accordion>
</Accordions>

## Quota-related questions

<Accordions>
  <Accordion title="What is quota and how is it calculated?">
    Quota is the internal unit AI Router uses to meter usage. Consumption for a request is derived from the number of prompt tokens and completion tokens, adjusted by the ratios configured for the model and group. The general shape is:

    ```
    Quota = Group Ratio × Model Ratio × (Prompt Tokens + Completion Tokens × Completion Ratio)
    ```

    The specific ratios depend on your deployment's configuration in the operation/model settings.

    <Callout type="warn">
      Pricing, currency, and rate values are TBD for this demo. Do not treat any number here as a real price. (pricing TBD)
    </Callout>
  </Accordion>

  <Accordion title="Why do I see 'insufficient quota' when my account quota looks fine?">
    Token quota and account quota are tracked separately:

    - **Account quota** is the balance for your whole account.
    - **Token quota** is an optional per-token (per-key) cap you set when creating a key.

    If a specific key has its own quota limit set, requests using that key can fail even when the account balance is sufficient. Check the per-token quota on the key you are using.
  </Accordion>
</Accordions>

## Channel configuration

<Accordions>
  <Accordion title="What are Weight and Priority on a channel?">
    These two parameters control routing among channels:

    - **Priority**: A higher number means higher priority; higher-priority channels are tried first.
    - **Weight**: Among channels that share the same priority, traffic is distributed in proportion to weight.

    For example, a priority-2 channel is used before a priority-1 channel. Among two priority-1 channels with weights 2 and 1, requests are distributed roughly 2:1.
  </Accordion>

  <Accordion title="I get 'no available channels' — what should I check?">
    Verify these settings line up:

    1. The user group assigned to your account.
    2. The group configured on the channel.
    3. The model list enabled on the channel (the requested model must be one the channel serves).

    A request only routes to a channel when the group and model both match.
  </Accordion>

  <Accordion title="Channel test error: invalid character '<' looking for beginning of value">
    This means the upstream returned an HTML page instead of JSON. Common causes are an upstream error page, an authentication/redirect page, or a network filter sitting between your deployment and the provider. Confirm the upstream URL, key, and network egress are correct.
  </Accordion>

  <Accordion title="Channel test error: ratio or price not configured">
    Configure the model's ratio or price in the operation/model settings before using the channel, or enable the relevant self-use mode if your deployment offers one. Until a model has a configured ratio, requests for it cannot be metered.
  </Accordion>
</Accordions>

## Connecting clients and deployment

<Accordions>
  <Accordion title="My OpenAI-compatible client shows 'Failed to fetch'">
    Check the following:

    1. Set the base URL to your AI Router endpoint (for example `https://ai.sammier.com/v1`) and avoid leaving a conflicting base URL elsewhere in the client.
    2. Confirm the API address and API key are entered correctly.
    3. If your site is served over HTTPS, make sure the API endpoint is also HTTPS — browsers block plain HTTP requests made from an HTTPS page.
  </Accordion>

  <Accordion title="Error: upstream returned 429 (rate limited)">
    A 429 means an upstream provider is temporarily rate-limiting requests (too many requests). Retry after a short delay, spread traffic across additional channels, or reduce request rate. General rate-limit behavior depends on the upstream provider's limits.
  </Accordion>

  <Accordion title="Will my data be lost when I upgrade?">
    It depends on how your database is stored:

    - **External database (e.g. MySQL/Postgres)**: Data persists across upgrades.
    - **SQLite**: Persist the database file on a mounted volume per your deployment instructions; otherwise data is lost when the container is recreated.

    Schema changes are generally applied automatically on startup. If a release needs manual steps, those will be called out in its notes.
  </Accordion>
</Accordions>
