## What AI Router is

AI Router is a unified, OpenAI-compatible AI gateway. It gives users **one API key and one base URL** to reach many model providers, with model groups, token groups, quotas, and usage tracking layered on top. It is an open-source project; the source lives at https://github.com/ssgzy/Router-new-api.

<Callout type="info" title="Demo status">
  AI Router is a brand-new demo. The notes below describe what enterprise use *could* look like. Anything marked **(TBD)** is not finalized — treat it as a placeholder, not a commitment.
</Callout>

## What enterprise use could look like

For a team or organization, a single gateway in front of many providers can reduce integration and management overhead. Typical goals:

<Cards>
  <Card title="One integration point" description="Point internal apps at a single OpenAI-compatible base URL instead of wiring each provider SDK separately." />
  <Card title="Centralized keys" description="Issue and rotate keys per team or service, and group them with token groups." />
  <Card title="Quotas & limits" description="Apply per-key quotas and daily limits so spend and load stay predictable." />
  <Card title="Usage tracking" description="See request and consumption records per key for internal cost attribution." />
</Cards>

### Connecting

Any OpenAI-compatible client works. Set the base URL and key, then call the standard endpoints.

<Tabs items={["Base URL", "Endpoint"]}>
  <Tab value="Base URL">
    ```
    https://ai.sammier.com
    ```
  </Tab>
  <Tab value="Endpoint">
    ```
    https://ai.sammier.com/v1
    ```
  </Tab>
</Tabs>

```bash
curl https://ai.sammier.com/v1/chat/completions \
  -H "Authorization: Bearer $CODE_ROUTER_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "your-model-group",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

The exact set of available models, model groups, and routing behavior depends on how a given deployment is configured.
