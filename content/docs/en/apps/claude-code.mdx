<Callout type="info" title="What is Claude Code?">
  Claude Code is Anthropic's terminal-based coding agent. It reads and edits files across a
  project, runs commands, and works directly from your shell.

  Because Claude Code honors standard Anthropic-compatible environment variables, you can point
  it at **Code Router** instead of Anthropic's default endpoint. Code Router then forwards your
  requests to whichever upstream model provider is configured behind your key.

  - Official Claude Code site: [https://www.anthropic.com/claude-code](https://www.anthropic.com/claude-code)
</Callout>

## How it works

Code Router exposes an OpenAI-compatible gateway (and an Anthropic-compatible surface for tools
like Claude Code). You get **one API key** and **one base URL**; Code Router resolves the
requested model to an upstream provider and tracks usage against your key.

To make Claude Code use Code Router, you set two environment variables before launching it:

| Variable | Value |
| --- | --- |
| `ANTHROPIC_BASE_URL` | `https://api.code-router.example` |
| `ANTHROPIC_AUTH_TOKEN` | Your Code Router API key (looks like `sk-...`) |

<Callout type="warn" title="Where do these values come from?">
  - The **base URL** above is a demo placeholder. Use the base URL of the Code Router deployment
    you actually connect to.
  - The **API key** is the key you create in your Code Router account. Treat it like a password.
  - Which underlying models are available, and any usage limits, depend on your Code Router
    deployment's configuration. (Available model list and limits: TBD / 待补)
</Callout>

## Prerequisites

<Steps>
<Step>
### A Code Router API key

Create a key in your Code Router account and copy it. It will look like `sk-...`.
</Step>
<Step>
### Node.js (recommended)

Claude Code runs on Node.js. Install the current LTS release from
[https://nodejs.org/](https://nodejs.org/) if you don't already have it. Verify with:

```bash
node --version
npm --version
```
</Step>
</Steps>

## Install Claude Code

<Tabs items={["macOS / Linux", "Windows"]}>
<Tab value="macOS / Linux">

Install via the official script:

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

If `claude` isn't found afterward, add the install directory to your `PATH`:

```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc && source ~/.bashrc
```

Verify the install:

```bash
claude --version
```

</Tab>
<Tab value="Windows">

On Windows, install Claude Code via npm in PowerShell:

```powershell
npm install -g @anthropic-ai/claude-code
```

<Callout type="info" title="Windows notes">
  - PowerShell is recommended over CMD.
  - If you hit permission errors, try running the terminal as Administrator.
  - If prompted that `~/.local/bin` is not on your `PATH`, add it:

    ```powershell
    [Environment]::SetEnvironmentVariable('Path', ([Environment]::GetEnvironmentVariable('Path','User') + ";$HOME\.local\bin"), 'User')
    ```
</Callout>

Verify the install:

```powershell
claude --version
```

</Tab>
</Tabs>

## Point Claude Code at Code Router

Set the two environment variables, then launch `claude`.

<Tabs items={["macOS / Linux", "Windows (PowerShell)"]}>
<Tab value="macOS / Linux">

Set the variables for your current shell session:

```bash
export ANTHROPIC_BASE_URL="https://api.code-router.example"
export ANTHROPIC_AUTH_TOKEN="sk-..."   # your Code Router API key
```

To make the change permanent, append those two `export` lines to your shell profile
(`~/.bashrc` or `~/.zshrc`), then reload it:

```bash
source ~/.bashrc   # or: source ~/.zshrc
```

</Tab>
<Tab value="Windows (PowerShell)">

Set the variables for the current session:

```powershell
$env:ANTHROPIC_BASE_URL = "https://api.code-router.example"
$env:ANTHROPIC_AUTH_TOKEN = "sk-..."   # your Code Router API key
```

To persist them across sessions:

```powershell
[Environment]::SetEnvironmentVariable('ANTHROPIC_BASE_URL', 'https://api.code-router.example', 'User')
[Environment]::SetEnvironmentVariable('ANTHROPIC_AUTH_TOKEN', 'sk-...', 'User')
```

Close and reopen PowerShell so the persisted values take effect.

</Tab>
</Tabs>

<Callout type="warn" title="This redirects all requests">
  Once `ANTHROPIC_BASE_URL` points at Code Router, Claude Code sends **all** model traffic
  through Code Router rather than Anthropic's default endpoint. Usage is billed/accounted against
  your Code Router key, not an Anthropic subscription.
</Callout>

## Run it

```bash
# Launch in the current directory
claude

# Or open a specific project
cd /path/to/your/project
claude
```

To choose a model from inside the session:

```bash
/model
```

Pick from the models your Code Router deployment exposes. The exact model names and which ones
are available depend on your deployment's configuration (model list: TBD / 待补).

## Verify the connection

A quick way to confirm Claude Code is talking to Code Router:

```bash
echo $ANTHROPIC_BASE_URL   # should print your Code Router base URL
claude --version           # confirms the CLI is installed
```

Then start `claude` and send a short prompt. If you get a normal response, routing is working.

You can also sanity-check the gateway directly with curl against the OpenAI-compatible endpoint:

```bash
curl https://api.code-router.example/v1/models \
  -H "Authorization: Bearer sk-..."
```

## Troubleshooting

<Accordions>
<Accordion title="`claude` command not found">
  The install directory isn't on your `PATH`. On macOS/Linux, add
  `export PATH="$HOME/.local/bin:$PATH"` to your shell profile and reload it. On Windows, add
  `~/.local/bin` to your user `Path` as shown in the install step, then reopen the terminal.
</Accordion>

<Accordion title="Environment variables not taking effect">
  - Confirm you edited the right profile (`.bashrc` vs `.zshrc`) and ran `source` on it, or that
    you reopened PowerShell after persisting them.
  - Print the value to check: `echo $ANTHROPIC_BASE_URL` (or `$env:ANTHROPIC_BASE_URL` in PowerShell).
  - Some shells cache exported values within a running session — open a fresh terminal.
</Accordion>

<Accordion title="Authentication / 401 errors">
  Make sure `ANTHROPIC_AUTH_TOKEN` holds a valid Code Router key (the `sk-...` value from your
  account), not an Anthropic key. Regenerate the key in Code Router if you suspect it was revoked.
</Accordion>

<Accordion title="Requests still go to the default endpoint">
  `ANTHROPIC_BASE_URL` may not be set in the shell where you launched `claude`. Set it in the same
  session (or persist it) and relaunch. Remember it must point at your Code Router base URL.
</Accordion>

<Accordion title="macOS blocks the binary from running">
  Open **System Settings → Privacy & Security**, then choose **Open Anyway** / **Allow** for the
  blocked item.
</Accordion>

<Accordion title="Linux: missing build dependencies">
  Some distributions need build tooling:

  ```bash
  # Ubuntu/Debian
  sudo apt install build-essential

  # CentOS/RHEL/Fedora
  sudo dnf groupinstall "Development Tools"
  ```
</Accordion>
</Accordions>
