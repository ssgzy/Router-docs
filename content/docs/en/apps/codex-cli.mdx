<Callout type="info" title="About this page">
  This guide shows how to point the OpenAI **Codex CLI** at **Code Router** so your
  Codex sessions route through your Code Router key and base URL.

  Code Router is a unified, OpenAI-compatible AI gateway: one API key plus one base
  URL lets you reach many model providers, with model groups, token groups, quotas,
  and usage tracking.

  - Codex CLI project: [https://github.com/openai/codex](https://github.com/openai/codex)
  - Code Router source: [https://github.com/ssgzy/Router-new-api](https://github.com/ssgzy/Router-new-api)
</Callout>

Codex CLI is a terminal-based coding agent that runs locally and edits code, applies
patches, and runs commands. Because it speaks the OpenAI-compatible protocol, you can
redirect it from the default OpenAI endpoint to Code Router by changing two settings:
the **base URL** and the **API key**.

## What you need

<Cards>
  <Card title="A Code Router API key" description="Create a key in the Code Router console. It looks like sk-... (key management TBD in your deployment)." />
  <Card title="The Code Router base URL" description="The OpenAI-compatible endpoint, e.g. https://ai.sammier.com/v1" />
  <Card title="Node.js 18+ (or Homebrew on macOS)" description="Required to install the Codex CLI via npm." />
</Cards>

<Callout type="warn" title="Use your own base URL">
  `https://ai.sammier.com` is a placeholder. Replace it everywhere below with
  the address of the Code Router instance you actually use — either your own
  self-hosted deployment or one provided to you. Do not connect API addresses or keys
  from unknown sources to a production environment.
</Callout>

## Configure Codex CLI for Code Router

Codex CLI reads its configuration from `~/.codex/config.toml` and takes the API key
from an environment variable. The pattern is the same on every platform:

1. Define a custom **model provider** that points `base_url` at your Code Router
   OpenAI-compatible endpoint.
2. Make that provider the default.
3. Export your Code Router key as an environment variable.

<Steps>
  <Step>
    ### Create or edit `~/.codex/config.toml`

    Add a provider block for Code Router and select it as the default. Replace the
    base URL and model with the ones available on your Code Router instance.

    ```toml
    model = "gpt-4o"            # any model your Code Router instance exposes
    model_provider = "code-router"

    [model_providers.code-router]
    name = "Code Router"
    base_url = "https://ai.sammier.com/v1"
    env_key = "CODE_ROUTER_API_KEY"
    ```

    The `env_key` value names the environment variable Codex will read the key from.
  </Step>

  <Step>
    ### Export your API key

    Set the environment variable named in `env_key` to your Code Router key.

    <Tabs items={["macOS / Linux", "Windows (PowerShell)"]}>
      <Tab value="macOS / Linux">
        ```bash
        export CODE_ROUTER_API_KEY="sk-..."
        ```

        Add the line to `~/.zshrc` or `~/.bashrc` to persist it across sessions.
      </Tab>
      <Tab value="Windows (PowerShell)">
        ```powershell
        setx CODE_ROUTER_API_KEY "sk-..."
        ```

        Open a new terminal afterward so the variable is picked up.
      </Tab>
    </Tabs>
  </Step>

  <Step>
    ### Run Codex

    Start Codex in any project directory:

    ```bash
    cd /path/to/your/project
    codex
    ```

    Use `/model` inside the session to switch between models your Code Router instance
    exposes.
  </Step>
</Steps>

<Callout type="info" title="Which models can I use?">
  The available model names depend on the model groups configured on your Code Router
  instance. Check the Code Router console for the list, and use those exact names in
  `config.toml` or via the `/model` command. (Specific model catalog and pricing: TBD.)
</Callout>

## Installing Codex CLI

If you do not yet have Codex CLI installed, follow the steps for your platform, then
return to the configuration section above.

<Tabs items={["macOS", "Linux", "Windows"]}>
  <Tab value="macOS">
    Install Node.js with Homebrew (see [https://brew.sh](https://brew.sh)), then
    install the CLI:

    ```bash
    brew update
    brew install node
    npm install -g @openai/codex
    codex --version
    ```

    <Callout type="info" title="Permission errors">
      If `npm install -g` fails on permissions, either prefix with `sudo`, or point
      npm at a user directory:

      ```bash
      npm config set prefix ~/.npm-global
      export PATH=~/.npm-global/bin:$PATH
      ```
    </Callout>
  </Tab>

  <Tab value="Linux">
    Install Node.js (LTS) and then the CLI:

    ```bash
    curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
    sudo apt-get install -y nodejs
    npm install -g @openai/codex
    codex --version
    ```

    <Callout type="info" title="Build dependencies">
      Some distributions need build tooling:

      ```bash
      # Ubuntu / Debian
      sudo apt install build-essential
      # Fedora / RHEL
      sudo dnf groupinstall "Development Tools"
      ```
    </Callout>
  </Tab>

  <Tab value="Windows">
    For the best experience on Windows, use the Windows Subsystem for Linux (WSL2),
    then follow the Linux steps inside WSL.

    ```powershell
    wsl --install
    ```

    Restart Windows after installation, open a WSL shell, install Node.js (an LTS
    release such as 22), then install the CLI:

    ```bash
    wsl
    nvm install 22        # if using nvm
    npm install -g @openai/codex
    codex --version
    ```

    <Callout type="warn" title="Windows notes">
      - Prefer PowerShell over CMD when not using WSL.
      - If you hit permission issues, try running the terminal as administrator.
      - Some antivirus tools may flag the install; add an exception if needed.
    </Callout>
  </Tab>
</Tabs>

## How routing works after configuration

Once the base URL points at Code Router, **every** model request from Codex — including
names that look like vendor-preset models — is sent to your configured Code Router
endpoint and authenticated with your Code Router key. Code Router then resolves the
model to a provider channel and tracks usage against your quota.

<Accordions>
  <Accordion title="Codex can't reach the server">
    Verify the `base_url` in `~/.codex/config.toml` ends with `/v1` and is reachable
    from your machine. A quick check:

    ```bash
    curl https://ai.sammier.com/v1/models \
      -H "Authorization: Bearer $CODE_ROUTER_API_KEY"
    ```
  </Accordion>
  <Accordion title="401 / authentication errors">
    Confirm the environment variable named by `env_key` is set in the current shell
    and that its value is a valid Code Router key (`sk-...`). Re-open the terminal
    after setting it.
  </Accordion>
  <Accordion title="Model not found">
    The model name must match one your Code Router instance exposes. List available
    models in the Code Router console and update `model` in `config.toml` accordingly.
  </Accordion>
</Accordions>
